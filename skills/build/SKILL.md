# Build

Use during implementation.

- Keep changes scoped to the approved spec or issue.
- Reuse existing files and patterns before adding new structure.
- Implement in small reviewable steps.
- Preserve backward compatibility unless the spec requires a breaking change.
- Do not revert unrelated user changes.
- Add or update focused tests for changed behavior.
- Record the validation command and result.
- Stop and ask when implementation choices would change scope or compatibility.
