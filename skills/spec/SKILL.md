# Spec

Use before implementation.

- Define the task in one clear outcome.
- State in-scope and out-of-scope work.
- List constraints, existing policies, and compatibility limits.
- Write acceptance criteria that can be verified.
- Check ambiguity before choosing an implementation path.
- Stop and ask when different interpretations would change the result.
