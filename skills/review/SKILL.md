# Review

Use before merge.

- Review correctness, risk, maintainability, and policy compliance.
- Check that the implementation matches the spec and acceptance criteria.
- Look for regressions, missing validation, and unrelated changes.
- Flag security, privacy, permission, or data-handling risks.
- Give evidence-based findings with file paths, commands, or observed behavior.
- Avoid vague comments that cannot be acted on.
