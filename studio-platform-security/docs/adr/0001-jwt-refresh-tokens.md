# ADR 0001: JWT + Refresh Token Persistence

## 상태
승인

## 맥락
무상태 인증(JWT)은 확장성에 유리하지만, 로그아웃/회수/세션 수명 관리가 어렵다.
보안 요구사항을 충족하려면 토큰 회전과 강제 폐기가 필요하다.

## 결정
- Access Token은 JWT로 발급한다.
- Refresh Token은 DB에 저장하고 회전/폐기 정책을 적용한다.
- 비밀번호 재설정 토큰도 DB 저장소를 사용한다.

## 결과
- 토큰 무효화/회전이 가능해 보안성이 향상된다.
- 다중 디바이스/세션 관리 정책을 구현할 수 있다.
- 운영자가 감사/잠금/정책 적용을 중앙에서 제어할 수 있다.
